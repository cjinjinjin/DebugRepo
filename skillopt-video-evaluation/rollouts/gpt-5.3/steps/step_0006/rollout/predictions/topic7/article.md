## 1. OpenClaw 是什么

- 运行在本地机器或 VPS，连接 WhatsApp/Telegram/飞书等多入口。
- 能力不止问答，还包括文件、邮件、代码、浏览器、工作流操作。
- 一句话：消息入口与工具链之间的 Agent 运行时与调度网关。

### 1.1 推理服务
- 大模型是参数文件，需通过推理服务加载到显存并提供 API。
- 推理服务通常无状态，请求结束后不保留上下文。

### 1.2 Memory
- 由于上下文窗口和成本限制，需要“短期原文 + 长期摘要/实体”分层记忆。
- 新请求前先组装记忆片段，再送入大模型。

### 1.3 RAG
- 解决知识冻结和幻觉问题：先检索，再生成。

### 1.4 MCP + Skills
- MCP 解决“能否调用工具”。
- Skills 解决“何时调用、按何顺序调用、失败如何回退”。

- 项目由 Peter Steinberger 发起，2025 年 11 月原型上线。

## 3. 部署与模型接入主线

- 环境建议：Node.js 20+，本地或云端均可。
- 实操顺序：安装 → 初始化向导 → 通道配置 → 任务验证。
- 模型参数重点：temperature、top_p、penalty、max_tokens。
- 工程建议：主模型 + fallback 备份模型，避免主链路中断。

## 4. 飞书与技能系统

- 飞书接入典型流程：建应用、开权限、发布、配置通道、事件订阅、联调。
- Skills 是“可复用 SOP 模块”，目标是把高频任务流程化。
- 支持内置技能与社区技能（ClawHub）扩展。

## 5. 安全指南（高优先级）

- 最小权限：默认拒绝不必要能力。
- 高风险动作需明确确认：支付、删除、安装、外发数据。
- 供应链安全：技能/插件输出不默认可信，混淆或不可审计内容先停。
- 凭证治理：密钥不进日志/聊天/提交，敏感目录默认禁读。
- 可疑行为处理：先停止执行，解释风险，再给替代方案。
