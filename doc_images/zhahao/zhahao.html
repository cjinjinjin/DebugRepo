Custom Model

Step1: Create folders to place your resources

Folder structure:

abo-models/[project_name]/[model_name]/m/[sub_model_name]/1(version)

Example: abo-models/picasso/basnet/m/basnet/1

Step2: Build your model

After pulling down the repo, create a container to customize your model.

Step 2.1: Container Set Up

Command:

docker run --gpus '"device=1"' --network=host -it --shm-size=1g --ulimit memlock=-1 --ulimit stack=67108864 -m 30g --cpuset-cpus 10-12 [image_name:tag] /bin/bash

 

Inside the container, the final project structure should be like this:

 

- abo-models/picasso/basnet/m/basnet

-- [version folder]

---- model.py

---- [other related files / folders to support model.py]

-- config.pbtxt

-- config.pbtxt.tp

-- python_env_package

Step 2.2: Build up model.py

Python script named ˇ°model.pyˇ± will be the entry of the model. You may copy one from other projects. This file should be put under your version folder ¨C for example, abo-models/picasso/basnet/m/basnet/1/model.py.

 

An Example Built by Hao Previously : https://msasg.visualstudio.com/Bing_Ads/_git/abo-models?path=/team/picasso/roi/m/roi/1/model.py 

 

And inside ˇ°model.pyˇ±, you should:

1.     Implement your model logic in "class ModelImpl ".

2.    Add unit test codes for offline debug / test (Optional)

3.    Update the checkpoint path to fit your need (Optional)

4.     NOT change anything in "TritonPythonModel" class.

 

Step 2.3: Other Related Scripts / Packages / Checkpoints

Place your other python codes / checkpoints under version folder.

Step 2.4: Python Environment Package

You can copy our environment directly and customize based on it: py38_roi.tar.gz - Repos

This env contains pytorch2.0, opencv4.2, and other common packages to support python3.8 running. If you don't need any other packages, use this package directly.

 

If you want to add some packages into it, here are some steps you can refer to:

1.     cd /tmp (or other path to save the tar temporarily)

2.     tar -xzf /path/to/your/[env_name].tar.gz -C /tmp

3.     pip install -t /tmp/lib/python3.8/site-packages [packages]

ˇ¤         Please check the pip version, it should be python3.8's pip.

ˇ¤         which pip

ˇ¤         pip --version

ˇ¤         find / -name pip to see all pip command

ˇ¤         If you want to update some packages, add "--upgrade" after pip command.

4.     You can check if packages were installed successfully via:

/tmp/bin/python import [package] [package] --version

5.     tar -czf [env_name].tar.gz -C /tmp .

Step4: Offline Test

Once custom model build finished, you should test offline to verify everything is expected.

Step 4.1: Setup Env Params

HereˇŻs an example, you should refer this to set up your own (you may not change the content for red part):

export ListeningPort=8886

export AB_MAX_SEQ_LEN=16

export AB_MODEL_PATH=picasso/lama/m

export AB_INSTANCE_GROUP_COUNT=1

export AB_ENTRYPOINT=/v2/models/lama/versions/1/infer

export AB_EXTRA_PARAMS="--allow-metrics false"

(Optional) if you find issues to setup environment, try this: export LD_LIBRARY_PATH=/usr/local/cuda/compat/lib.real

After this command, use ˇ°runˇ± command to start the offline service. Then you can call localhost:[LsteningPort] with request bodies to test or debug.

HereˇŻs an example to send request: offline_test_request_script_example.py

 

Step5: Build Docker Image

If everything goes well, then you should build this container as a docker image package for online use.

Commands:

(inside container) exit

(outside container) docker stop [container_id]

(outside container) docker commit [container_id] [alias]:[model_name]-[version]

 

Then use ˇ®docker imagesˇŻ to check if your image was generated.
